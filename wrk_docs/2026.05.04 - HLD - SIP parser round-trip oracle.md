# HLD — SIP parser round-trip oracle

**Date:** 2026-05-04
**Author:** Claude (with Arthur)
**Status:** Draft for review
**Related:** `2026.05.03 - HLD - sip-parser-rewrite.md`,
`wrk_journals/2026.05.04 - JRN - parser-fuzz-run6-triage.md`

## Context

The differential oracle (`tests/parser_diff_oracle/mod.rs`) compares
our parser to `rsip 0.4` over a fixture corpus and a libfuzzer target.
Through M11 it has caught real bugs and pinned 17 rsip-narrowness
divergences. After this many pins the differential signal is decayed —
the remaining divergences are dominantly rsip's, not ours, and a
differential oracle is structurally blind to bugs both parsers agree
on (e.g. a header silently dropped by both, lossy serialization).

The next leverage point is a **round-trip oracle**: parse the wire
form, serialize it back, parse again, and assert the second parse
matches the first. This is a pure self-consistency check on our
parser/serializer pair.

## Goals

- Catch bugs where parse and serialize disagree (any non-fixed-point
  in the parse → serialize cycle).
- Run as both a static fixture suite and a libfuzzer target.
- Land on a new oracle without touching the differential one.

## Non-goals

- Catch bugs both parse and serialize *agree* on (silently dropped
  headers, character escaping that's wrong but symmetric). That needs
  a separate oracle (grammar conformance, hand-curated vectors).
- Replace the differential oracle. It still has value as a
  curated regression suite of known divergences; it is just no longer
  the right thing to fuzz heavily.

## What's already in place

- `Request::to_bytes`, `Response::to_bytes`, `Message::to_bytes`
  emit canonical wire form (long header names, `Name: value\r\n`,
  CRLF blank line, Content-Length rewritten to `body.len()`).
- `Message`, `Request`, `Response`, `Headers`, `Header` all
  derive `PartialEq, Eq`.
- Four unit tests prove round-trip works on simple inputs
  (`test_message_round_trip_*`, `test_round_trip_with_mismatched_content_length`).

So the *primitives* exist. What's missing is corpus coverage and a
fuzz target.

## Design

### The invariant

The first round-trip is allowed to **normalize** (compact → long
header names, whitespace collapse, stale Content-Length → real
length, etc.). After one normalization, parse/serialize must be a
**fixed point**. Concretely:

```
m1 = Message::parse(input)?           // OK is required to proceed
m2 = Message::parse(&m1.to_bytes())?  // must succeed
m3 = Message::parse(&m2.to_bytes())?  // must succeed
assert_eq!(m2, m3);                   // AST-equal — the fixed point
assert_eq!(m2.to_bytes(), m3.to_bytes()); // bytes-equal — stronger guard
```

If the first parse fails, round-trip is undefined; the oracle returns
without asserting. (Fuzzing produces overwhelmingly malformed inputs;
this is the right discard rule.)

#### Why "fixed point at m2", not "m1 == m2"?

Because the first round-trip legitimately normalizes — comparing
`m1 == m2` would force every fixture to already be in canonical form,
which is a needless corpus restriction. The fixed-point form catches
the same class of bugs (any non-idempotency) without that restriction.
Known normalizations covered (already proven by existing unit tests):

- Compact header names → long names.
- Whitespace after `:` → single space.
- Folded header values → joined into one line.
- Stale Content-Length → rewritten to real body length.

### Oracle module

New file `crates/rsiprtp/tests/parser_roundtrip_oracle/mod.rs`,
parallel to the existing `parser_diff_oracle/mod.rs`. One public
function:

```rust
/// Assert the round-trip fixed-point invariant on `bytes`.
/// If first parse fails, returns silently — round-trip is undefined.
pub fn assert_roundtrip_fixed_point(bytes: &[u8]) { ... }
```

No skips, no configuration, no feature flags. ~30 LOC total.

### Static-fixture driver

New file `crates/rsiprtp/tests/parser_roundtrip.rs`, parallel to
`tests/parser_diff.rs`. One `#[test]` per fixture in the existing
`tests/fixtures/{mdsiprtp3,handcrafted}/` directories. Mechanical
mirror of the differential driver — same fixtures, different
oracle. ~30 LOC including imports.

Fixtures are reused as-is; no new corpus needed at M11. Adding
fixtures later is mechanical.

### Fuzz target

New file `fuzz/fuzz_targets/sip_message_roundtrip.rs`, mirror of
`sip_message_parse_diff.rs`. Calls
`assert_roundtrip_fixed_point(data)` on arbitrary bytes. ~15 LOC.

Shares the oracle module via `#[path = ...]` the same way the
differential fuzz target shares its oracle.

### Skip / pin mechanism

**None at start.** Round-trip is self-consistency on code we own —
any failure is a real bug. If a defensible non-fixed-point case turns
up (an honest design trade-off where round-trip can't be a fixed
point), we add a pinned skip then, with the same pattern as the
differential oracle's `Skip::Reason` enum. Reactively, not up front.

## Trade-offs and acknowledged risks

**Risk:** the oracle might find no bugs at all if our parser/serializer
pair is already a fixed point on real inputs. That's a *good* outcome
but means the oracle's value is regression-prevention, not initial
discovery. Acceptable — it's still cheap to build and run.

**Risk:** the corpus is small (existing fixtures only). Mitigated by
the fuzz target, which generates novel inputs. If fuzz finds nothing
in a 10-minute run, we have moderate confidence the invariant holds
broadly; if it finds something, we triage and pin like we did for the
differential oracle.

**Trade-off rejected:** I considered a stricter "bytes-equal after
first round-trip" oracle. This would force every fixture to be
already-canonical, which the differential corpus is not. The
fixed-point-after-one-normalize variant catches the same bug class
without that restriction.

**Trade-off rejected:** I considered building round-trip *into* the
differential oracle as a third assertion. Mixing two independent
invariants in one function makes failures harder to diagnose
(differential failure or round-trip failure?). Separate oracle,
separate failure mode.

## Out of scope

- Grammar-conformance oracle (catches bugs both parse and serialize
  agree on). Future work.
- Hand-curated AST vectors for parse output. Future work.
- Replacing the differential oracle. Stays as a regression suite.
- Any change to the parser, serializer, or existing tests.

## Implementation plan

1. Write `parser_roundtrip_oracle/mod.rs` (~30 LOC).
2. Write `tests/parser_roundtrip.rs` with one test per existing
   fixture (~30 LOC).
3. Write `fuzz/fuzz_targets/sip_message_roundtrip.rs` (~15 LOC).
4. Add the new fuzz target name to `fuzz/Cargo.toml`.
5. `cargo test --workspace` — confirm fixture suite passes.
6. `cargo fuzz run sip_message_roundtrip -- -max_total_time=600`
   — 10-minute smoke fuzz. Triage findings into journal pins as
   they arise.

Total estimated diff: ~75 LOC across 3 new files + 1 Cargo.toml line.
No changes to existing code.
