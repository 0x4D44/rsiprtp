## Multi-stage plan to reach 100%
### Stage 1: Pinpoint misses (diagnostics)
- Re-run `cargo +nightly llvm-cov --branch --summary-only --workspace --exclude gabby --ignore-run-fail -- --test-threads=1` after each change.
- Use `target/llvm-cov/coverage.json`, `coverage.lcov`, and `coverage-inst.txt` to map any remaining region misses to specific line spans.
- If the mismatch persists without identifiable missing lines, document likely llvm-cov accounting issues.

### Stage 2: Targeted tests for edge paths
- Add tests for missing-header and parse-error paths in `crates/mdsiprtp-sip/src/message.rs` (e.g., `from_tag()` and `from_tag_and_uri()` missing header cases for request and response).
- Add focused manager tests for any remaining unexercised paths (e.g., `handle_message` response dispatch, inconsistent handle/type maps) in `crates/mdsiprtp/tests/coverage_instantiations.rs` or `crates/mdsiprtp-transaction/src/manager.rs`.
- Keep tests small, isolated, and deterministic (AAA pattern).

### Stage 3: Iterate to 100%
- Re-run coverage after each batch of tests.
- If 100% remains blocked, evaluate lightweight code adjustments to reduce uncovered region counts without changing runtime behavior (only with explicit approval).
- Finalize with a clean coverage snapshot and a short report of any residual tool limitations.
