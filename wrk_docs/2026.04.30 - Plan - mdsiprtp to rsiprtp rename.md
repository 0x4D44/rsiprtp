# 2026.04.30 - Plan - mdsiprtp to rsiprtp rename

## Goal

Rename the workspace from `mdsiprtp` to `rsiprtp` across all 12 crates (the
facade `mdsiprtp` plus 11 internal `mdsiprtp-*` crates), aligning the source
with the published name reserved on crates.io. `gabby` keeps its name. Keep
the multi-crate workspace structure for compile-time architectural enforcement
and incremental-rebuild wins (per Arthur's call).

## Non-goals

- Not collapsing the workspace into a single crate.
- Not modifying `wrk_journals/` or `wrk_docs/` content — those are historical
  record and reference the old name correctly for their date.
- Not bumping the workspace version (currently `0.2.0`); first published
  version will be `0.2.0` over the existing `rsiprtp v0.0.0` placeholder.
- Not setting up `cargo-fuzz`, `missing_docs`, `cargo-deny`, or MSRV in this
  pass — separate sessions.
- Not reserving `rsiprtp-*` names on crates.io. Internal crates get
  `publish = false`; squatting risk on those names is acceptable.
- Not editing CHANGELOG (we don't have one yet).

## Scope by file count

`grep mdsiprtp` shows 931 occurrences across 126 files. After excluding
`wrk_journals/` (~50 occurrences across ~10 files) and `wrk_docs/` (~250 across
~20 files), the rename touches roughly **96 files**:

- 13 `Cargo.toml`s (workspace + 12 crates)
- ~70 `.rs` files (source + tests + examples)
- 2 `.yml` (one CI workflow + docker-compose)
- 5 docker config files (`Dockerfile`, `pjsip.conf`, `rtp.conf`,
  `extensions.conf`, plus `docker-compose.yml`)
- 5 docs (`README.md`, `CLAUDE.md`, `CONTRIBUTING.md`, `SECURITY.md`,
  `crates/gabby/README.md`)
- `Cargo.lock` (auto-regenerated by cargo, not edited manually)

## Execution order

The intermediate states between steps will not compile. That's expected; we
land everything in one commit.

1. **Workspace `Cargo.toml`** — update `members`, `default-members`, and the
   keys+paths under `[workspace.dependencies]`.
2. **`git mv crates/mdsiprtp* crates/rsiprtp*`** — 12 directory renames. Use
   `git mv` so rename detection survives in `git log --follow`.
3. **Per-crate `Cargo.toml`** — update `name`, `description`, and dependency
   keys (`mdsiprtp-core.workspace = true` → `rsiprtp-core.workspace = true`).
   Add `publish = false` to all 11 internal crates (the facade `rsiprtp`
   stays publishable).
4. **`gabby/Cargo.toml`** — `mdsiprtp = { path = "../mdsiprtp" }` →
   `rsiprtp = { path = "../rsiprtp" }`.
5. **Source-wide `sed`** — replace `mdsiprtp` → `rsiprtp` everywhere except
   `wrk_journals/`, `wrk_docs/`, `Cargo.lock`, `target/`, `.git/`. Pattern is
   unambiguous: every `mdsiprtp-` (kebab) and `mdsiprtp_` (snake) wants the
   replacement. The handful of literal-string occurrences (SDP `tool:` value
   in tests, example session_name) are also worth changing for cohesion.
6. **Cargo.lock regeneration** — `cargo build --workspace --exclude gabby`
   will rebuild `Cargo.lock` against the new package names. Don't edit it
   manually.
7. **Validation gates:**
   - `cargo fmt --check`
   - `cargo build --workspace --exclude gabby`
   - `cargo test --workspace --exclude gabby`
   - `cargo clippy --workspace --exclude gabby --all-targets -- -D warnings`
   - `cargo doc --workspace --exclude gabby --no-deps`
   - `cargo build -p rsiprtp --examples`
8. **Commit + push** — single atomic commit `Rename mdsiprtp to rsiprtp
   across workspace`. Arthur authorized push for this task.

## What we might be missing — flagged risks

1. **Internal-crate squatting on crates.io.** After publishing `rsiprtp v0.x`,
   nothing stops a third party from registering `rsiprtp-core`, `rsiprtp-sip`,
   etc. Since those won't be published from this repo (`publish = false`), a
   collision wouldn't break us, but it could be confusing. Defensive option
   not taken: publish 0.0.0 placeholders for each. Skipping unless requested.

2. **Pre-existing TCP test flakiness.** Two tests
   (`tcp::tests::test_tcp_bind`, `test_tcp_bind_forced_local_addr_error`)
   intermittently fail per the earlier history-scrub validation run. Post-
   rename test results should match pre-rename: same flakes, no new failures.
   I'll note any new failures explicitly.

3. **Doctests in `lib.rs` files.** Several `//! \`\`\`rust,ignore` blocks
   import `mdsiprtp::prelude::*`. The `,ignore` flag means cargo doesn't
   compile them, so they wouldn't break the build even if missed — but
   `sed` will catch them anyway since they live in `.rs` files.

4. **GitHub repo redirect.** The repo was renamed `mdsiprtp` → `rsiprtp` on
   GitHub already; old clones that still have origin pointing at
   `github.com/0x4D44/mdsiprtp` will keep working via redirect, but the
   redirect isn't permanent. Anyone who cloned should run
   `git remote set-url origin https://github.com/0x4D44/rsiprtp.git`.

5. **Concurrent agent.** Repo is currently clean and the other agent's most
   recent commit was `8d3fc5c` (CongestionController docs). Will recheck
   `git status` immediately before the rename. If they push something mid-
   execution, will rebase before the final push.

6. **Push rejection.** If origin/main has moved between local commits and
   push, push fails. Will pull --rebase rather than force.

7. **Line endings.** Working on Windows; git is configured for `core.autocrlf`
   on this checkout (saw "LF will be replaced by CRLF" warnings during the
   licensing commits). `sed -i` should preserve existing line endings since
   it doesn't touch the EOLs. Will spot-check with `git diff --stat` after.

## Tooling

- `git mv` for directory renames (preserves history).
- `bash` + `sed -i` for source-wide replacement, scoped via `git ls-files`
  with the wrk_journals/wrk_docs/Cargo.lock exclusion.
- `cargo build/test/fmt/clippy/doc` for validation.
- `cargo metadata --no-deps --format-version=1 | jq '.packages[].name'` for a
  post-rename sanity check that all package names are correct.

## Rollback

Single-commit landing means rollback is `git reset --hard HEAD~1` (before
push) or `git revert <sha>` (after push). I'll keep the working repo's
pre-rename HEAD recoverable via reflog at minimum.
