## Goal

Delete `crates/rsiprtp/src/srtp/dtls.rs` (990 lines, no working handshake)
and every reference to it. After this lands, `rsiprtp::srtp` advertises
only what it does: SDES key exchange (RFC 4568) feeding `SrtpContext`.

## Decision

**Option A (full removal), not A.5.** No internal caller uses any
`srtp::dtls` type, so keeping `Fingerprint` / `SrtpProfile` for
hypothetical external consumers is exactly the kind of premature
abstraction CLAUDE.md warns against. Re-adding from git history if ever
needed is trivial.

## Versioning — bump to **0.3.0**, not 0.2.1

Cargo's pre-1.0 SemVer rule treats `0.x.y → 0.x.(y+1)` as **automatically
compatible**: anyone with `rsiprtp = "0.2"` would silently absorb the
removal on the next `cargo update` and hit a compile error. The brief
flagged this risk explicitly ("anyone who pinned 0.2 expecting DTLS
support") — V1 of this HLD waved it away; V2/V3 corrects it. Pre-1.0
still has rules — the **minor** is the breaking-change axis.

The workspace version lives in `d:\language\rsiprtp\Cargo.toml:15`
(`[workspace.package]`); the per-crate manifest is `version.workspace =
true` and is *not* the file to edit.

## Out of scope

- Implementing real DTLS-SRTP (Option B in the brief). Future brief + HLD.
- Touching `srtp::sdes`, `srtp::context`, `srtp::kdf` — unrelated, working.
- Refactoring `security.rs` integration tests beyond the one comment.
- Dependency pruning. Verified: `sha1` (used by `srtp/context.rs`,
  `ice/agent.rs`, `ice/turn.rs`), `sha2` (used by `sip/auth.rs`),
  `thiserror` (used by 13 files) all retain non-DTLS callers.

## Public API impact

Removes the entire `rsiprtp::srtp::dtls` module. The 11 items inside are:
8 types (`DtlsConfig`, `DtlsError`, `DtlsRole`, `DtlsSrtpKeys`,
`DtlsState`, `Fingerprint`, `FingerprintHash`, `SrtpProfile`) reachable
via both `rsiprtp::srtp::dtls::…` and `rsiprtp::srtp::…` (re-exported in
`srtp/mod.rs`); plus 3 free functions (`extract_srtp_keys`,
`parse_use_srtp_extension`, `build_use_srtp_extension`) reachable only
via the full `rsiprtp::srtp::dtls::…` path. All 11 disappear when step 1
deletes the file and step 2 removes the `pub mod dtls;` and `pub use`
lines.
Surviving `rsiprtp::srtp` surface: `SrtpContext`, `SrtcpContext`,
`CryptoSuite`, `SessionKeys`, `SdesAttribute`.
No `Deprecated` step — dev sandbox, clean delete.
