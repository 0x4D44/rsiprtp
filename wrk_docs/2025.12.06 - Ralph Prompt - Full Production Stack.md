## Objective

Complete the mdsiprtp SIP/RTP stack to full production quality, implementing ALL planned phases plus additional features required for real-world deployment. The stack must successfully:

1. Register with Asterisk PBX
2. Make outbound calls with two-way audio
3. Receive inbound calls with two-way audio
4. Work behind NAT using ICE
5. Encrypt all media with SRTP
6. Support conferencing with audio mixing

## Implementation Phases

### PHASE 1 COMPLETION: End-to-End Integration

**Missing from Phase 1:**
- [ ] RTCP sender/receiver reports
- [ ] SIP REGISTER method support
- [ ] Digest authentication (RFC 2617)
- [ ] Integration example that makes a real call

**Deliverables:**
1. `mdsiprtp-rtp`: Add RTCP packet parsing/building, sender reports, receiver reports
2. `mdsiprtp-sip`: Add REGISTER request building, WWW-Authenticate parsing, Authorization header building
3. `mdsiprtp-session`: Add `RegistrationManager` for periodic re-registration
4. `examples/basic_call.rs`: Working example that registers and calls Asterisk echo test

**Test Criteria:**
- Register with Asterisk (sip.conf or pjsip.conf)
- Call *43 (echo test) and hear audio echoed back
- Receive call from softphone (Obok/Obok Obok or Obok/Obok Obok Obok Obok:Obok Obok)

### PHASE 2: TCP/TLS Transport & Codecs

**Tasks:**
- [ ] TCP transport with connection management
- [ ] TLS transport using rustls
- [ ] Transport selection via DNS SRV (RFC 3263)
- [ ] G.722 wideband codec
- [ ] Opus codec (via audiopus FFI)
- [ ] Codec preference negotiation

**New Crates/Modules:**
1. `mdsiprtp-transport/src/tcp.rs` - TCP with framing (Content-Length)
2. `mdsiprtp-transport/src/tls.rs` - TLS wrapper using tokio-rustls
3. `mdsiprtp-transport/src/resolver.rs` - DNS NAPTR/SRV resolution
4. `mdsiprtp-media/src/codec/g722.rs` - G.722 using ezk-g722
5. `mdsiprtp-media/src/codec/opus.rs` - Opus using audiopus

**Test Criteria:**
- Connect to Asterisk over TLS
- Call with Opus codec selected
- DNS SRV resolution for _sip._udp.example.com

---

### PHASE 3: Media Security & NAT Traversal

**Tasks:**
- [ ] SRTP encryption/decryption (AES-CM, HMAC-SHA1)
- [ ] SDES key exchange (a=crypto in SDP)
- [ ] DTLS-SRTP key exchange
- [ ] STUN binding requests/responses
- [ ] ICE candidate gathering (host, srflx, relay)
- [ ] ICE connectivity checks
- [ ] TURN relay allocation

**New Crates:**
1. `mdsiprtp-srtp/` - SRTP using webrtc-srtp or custom impl
2. `mdsiprtp-ice/` - ICE using webrtc-ice or custom impl

**SDP Additions:**
- `a=crypto:` lines for SDES
- `a=fingerprint:` for DTLS
- `a=ice-ufrag:`, `a=ice-pwd:`
- `a=candidate:` lines

**Test Criteria:**
- SRTP call to Obok/Obok Obok (requires encryption)
- Call works from behind NAT using STUN
- Call works using TURN relay when STUN fails

---

### PHASE 4: Advanced Features & Applications

**Call Features:**
- [ ] Call hold/resume (re-INVITE with a=sendonly/recvonly)
- [ ] Attended call transfer (REFER with Replaces)
- [ ] Blind call transfer (REFER)
- [ ] DTMF send/receive (RFC 4733 telephone-event)
- [ ] Message Waiting Indication (SUBSCRIBE/NOTIFY)

**Conferencing:**
- [ ] N-way audio mixing with CSRC tracking
- [ ] Active speaker detection (energy-based)
- [ ] N-1 mixing (each participant hears others only)

**Media Features:**
- [ ] Audio recording to WAV file
- [ ] Audio playback from WAV file
- [ ] Silence detection (VAD)
- [ ] Comfort noise generation (RFC 3389)

**Applications:**
1. `apps/voicemail/` - Voicemail application
   - Answer calls
   - Play greeting
   - Record message
   - DTMF to skip/delete

2. `apps/ai-bridge/` - AI agent call bridge
   - Bridge caller to AI backend
   - Mix AI audio with caller
   - Support barge-in
   - Websocket connection to AI

**Test Criteria:**
- 3-party conference with mixing
- Transfer call between extensions
- Voicemail records and plays back messages
- AI bridge connects to mock AI backend
