# HLD — Collapse workspace into single `rsiprtp` crate

**Date:** 2026-05-01
**Author:** Arthur + Claude
**Status:** Proposed

## Goal

Make `rsiprtp` a single self-contained crate on crates.io while keeping the
workspace useful for development. Today the facade `pub use`s nine internal
crates that aren't (and won't be) published, so `cargo publish -p rsiprtp`
would produce a tarball that no downstream user can build.

## Current state

```
workspace
├── crates/rsiprtp           (facade, depends on 9 internal crates — to be published)
├── crates/rsiprtp-core
├── crates/rsiprtp-sip
├── crates/rsiprtp-transaction
├── crates/rsiprtp-dialog
├── crates/rsiprtp-transport
├── crates/rsiprtp-sdp
├── crates/rsiprtp-rtp
├── crates/rsiprtp-media     (feature: opus)
├── crates/rsiprtp-session   (forces media[opus])
├── crates/rsiprtp-srtp      (feature: dtls — gates openssl native dep)
├── crates/rsiprtp-ice
└── crates/gabby             (publish=false, depends on rsiprtp via path)
fuzz/  (standalone workspace; depends on rsiprtp-rtp/-sip/-sdp/-srtp/-ice via path)
```

Internal dep graph is a clean DAG; no external consumers. `gabby` uses only
the `rsiprtp::*` public surface (one import line, `rsiprtp::sip::...`).
`srtp` and `ice` are not surfaced by the facade today; they're referenced
only by `fuzz/`.

## Target state

```
workspace
├── crates/rsiprtp           (PUBLISHED — every former internal crate is now a module)
└── crates/gabby             (publish=false, unchanged)
fuzz/  (standalone workspace; one rsiprtp path-dep with dtls feature)
```

`crates/rsiprtp/src/lib.rs`:

```rust
#![warn(missing_docs)]   // demoted from deny — see audit finding below
pub mod core;
pub mod sip;
pub mod transaction;
pub mod dialog;
pub mod transport;
pub mod sdp;
pub mod rtp;
pub mod srtp;            // pure crypto path always available
pub mod ice;
pub mod media;
pub mod session;

pub mod prelude { /* unchanged set of common re-exports, paths rewritten */ }
```

Each module's source is a 1:1 move of the former crate's `src/` tree. Inline
`#[cfg(test)] mod tests` blocks come along unchanged. Sub-`tests/`
directories under `src/` (`sip/tests`, `transaction/{client,server}/tests`,
`sdp/tests`, `rtp/tests`) move with their parents.

## Key decisions

1. **One crate, internal modules — not `#[path]` tricks.** The published
   tarball is what end users build against; making it a real single crate
   is the only configuration that doesn't lie about what it is.

2. **Bring `srtp` and `ice` in as modules** of the published crate. Per
   directive: don't drop any features. SRTP encryption and ICE/STUN/TURN
   become reachable via `rsiprtp::srtp` and `rsiprtp::ice`.

3. **Keep one feature flag — `dtls` — on the published crate.** It gates
   `openssl`, a heavy native C dep that not every consumer wants on every
   target. This is the textbook legitimate use of a feature flag.
   Default-off, opt-in.

4. **Drop the `opus` feature flag from `rsiprtp-media`.** `ropus` is
   pure-Rust (per commit f8293a6) and `rsiprtp-session` already forces
   `media[opus]` on unconditionally. Making it always-on costs nothing
   and removes a flag.

5. **Lift two pieces of crate-level config to the workspace:**
   - `unexpected_cfgs` `check-cfg = ["cfg(coverage)", "cfg(coverage_nightly)"]`
     currently lives only on `rsiprtp-transaction` but `cfg(coverage)` is
     used in `rsiprtp-sip` and `gabby` too — pre-existing latent warnings
     today. Move to `[workspace.lints]`.
   - `hickory-resolver = "0.24"` is inline-pinned in `rsiprtp-transport`'s
     Cargo.toml. Lift to `[workspace.dependencies]` for consistency with
     the other deps.

6. **No deprecation shims, re-export crates, or compatibility façades.**
   Per the directive: development environment, no migration needed. Old
   crate names disappear cleanly from the workspace.

## Audit findings — what I verified before locking the design

These were checked against the live tree on 2026-05-01:

- **No external crates depend on the to-be-deleted internal crates.**
  `gabby` uses `rsiprtp::*` only; `fuzz` uses 5 internal crates directly
  but is the only consumer that needs rewriting.
- **No intra-doc rustdoc links of the form `` [`rsiprtp_core::X`] ``** exist
  anywhere in the codebase. Module path rewrites won't break docs.
- **No `build.rs` files** — nothing compiled at build time needs path
  awareness.
- **Only two existing feature flags** — `media/opus` and `srtp/dtls`. (3)
  and (4) cover both.
- **License files are already inside `crates/rsiprtp/`** (LICENSE-MIT,
  LICENSE-APACHE) and ship in the current tarball — no work needed.
- **README.md is shipped via `readme = "../../README.md"`**; cargo handles
  the `..` correctly. Verified via `cargo package --list`.
- **The current tarball ships ~30 test files including the entire
  baresip_integration framework** (Asterisk fixtures, scenario tests, etc.).
  This is bloat and shouldn't go to crates.io. The merged crate needs
  `package.exclude = ["tests/baresip_integration/**", "tests/baresip_integration.rs", "tests/asterisk_integration.rs"]` (or
  similar) to slim the published artifact. Pre-existing issue, but the
  collapse is the right moment to fix it because rsiprtp-session's
  integration tests will move into the same `tests/` dir.
- **`#![deny(missing_docs)]` on the facade vs `#![warn(missing_docs)]` on
  every internal crate** — the merged crate must use `warn`, not `deny`,
  unless we're prepared to write doc comments for every undocumented `pub`
  item across 11 former crates first. Treat the doc audit as a follow-up.
- **`public-api.txt` baseline checked in.** The collapse turns
  `pub use rsiprtp_core as core` into `pub mod core { ... }`. The
  rendered API surface in `cargo public-api` output will look quite
  different even though the items are identical. Regenerate the baseline
  in the same PR; expect a large mechanical diff.
- **`docs.rs` builds with `all-features = true`** (currently `dtls` is the
  only feature). docs.rs Linux builders have system openssl, so this works
  out of the box.

## Mechanics

Order matters because each step keeps the tree compiling.

1. **Move sources.** For each `X` in `{core, sip, transaction, dialog,
   transport, sdp, rtp, srtp, ice, media, session}`:
   - `crates/rsiprtp-X/src/lib.rs` → `crates/rsiprtp/src/X/mod.rs`
   - Everything else under `crates/rsiprtp-X/src/` → `crates/rsiprtp/src/X/`
     (preserve subtree shape: `sip/tests/`, `transaction/{client,server}/`,
     etc.).

2. **Move integration tests.** `crates/rsiprtp-session/tests/*.rs`
   (`bitrate_bridge_remb.rs`, `rtcp_drives_bridge.rs`) →
   `crates/rsiprtp/tests/`. Verified no name collisions with the existing
   tests in `crates/rsiprtp/tests/`.

3. **Rewrite imports across the moved sources.**
   - `rsiprtp_core::` → `crate::core::` (and so on for each former crate)
   - `use crate::X` *inside* a moved crate (e.g. `crate::error` inside
     rsiprtp-core's source) becomes `use crate::core::error` once the module
     re-roots under `core/`. This is the only mechanically large diff;
     `rg "^use crate::" crates/rsiprtp/src/` after the move will surface
     every reference that needs the new module prefix.

4. **Rewrite dependents.**
   - `gabby/Cargo.toml` is already correct (`rsiprtp = { path = "../rsiprtp" }`)
     and `gabby/src/` already uses `rsiprtp::*`. No source changes; only the
     workspace path stays the same.
   - `fuzz/Cargo.toml`: replace the 5 `rsiprtp-*` path-deps with one
     `rsiprtp = { path = "../crates/rsiprtp", features = ["dtls"] }` (the
     `dtls` feature is needed for `srtp_use_ext.rs`). Source files
     (`srtp_use_ext.rs`, `srtp_sdes.rs`, `ice_candidate.rs`, the rtp/sip/sdp
     fuzz targets) get import rewrites from `rsiprtp_X::` to `rsiprtp::X::`.

5. **Collapse `crates/rsiprtp/Cargo.toml` deps.** Replace the nine
   `rsiprtp-*.workspace = true` lines with the actual transitive deps:
   `rsip`, `thiserror`, `bytes`, `uuid`, `md5`, `sha2`, `sha1`, `hex`,
   `rand`, `tokio` (workspace `["full"]`), `tokio-rustls`, `rustls`,
   `rustls-pemfile`, `webpki-roots`, `hickory-resolver`,
   `audio-codec-algorithms`, `ezk-g722`, `ropus`, `dasp`, `crossbeam`,
   `aes`, `ctr`, `hmac`, `base64`, `tracing`, `serde`. The
   `[features] dtls = ["openssl"]` block carries over from `rsiprtp-srtp`.

6. **Merge dev-deps.** Union of every former crate's `[dev-dependencies]`
   moves to `crates/rsiprtp/[dev-dependencies]`. New entries:
   `rcgen`, `rsa`, `p256`, `tracing-subscriber`, `serde_json`, `tempfile`.

7. **Add `package.exclude`** to drop heavyweight test fixtures from the
   published tarball:
   ```toml
   [package]
   exclude = [
       "tests/baresip_integration/**",
       "tests/baresip_integration.rs",
       "tests/asterisk_integration.rs",
   ]
   ```

8. **Demote crate-level lint.** Change `#![deny(missing_docs)]` to
   `#![warn(missing_docs)]` in the new `crates/rsiprtp/src/lib.rs`.
   Re-tightening to `deny` is a follow-up after the doc audit.

9. **Lift workspace lints.** Add to root `Cargo.toml`:
   ```toml
   [workspace.lints.rust]
   unexpected_cfgs = { level = "warn", check-cfg = ["cfg(coverage)", "cfg(coverage_nightly)"] }
   ```
   and `[lints] workspace = true` in `crates/rsiprtp/Cargo.toml` and
   `crates/gabby/Cargo.toml`. Drop the per-crate `[lints.rust]` block from
   the (about-to-be-deleted) `rsiprtp-transaction/Cargo.toml`.

10. **Remove deleted crates from `[workspace] members`** and from
    `[workspace.dependencies]`. Workspace members shrink to
    `["crates/rsiprtp", "crates/gabby"]`; default-members to
    `["crates/rsiprtp"]`.

11. **Delete the 11 empty crate dirs** once everything compiles:
    `rm -r crates/rsiprtp-{core,sip,transaction,dialog,transport,sdp,rtp,srtp,ice,media,session}`.

12. **Trim README.** The "ICE/STUN/TURN integration end-to-end through
    `CallManager`" line in "What's not included (yet)" stays accurate
    (low-level types are reachable but `CallManager` doesn't drive ICE).
    No README changes required for SRTP — it's reachable as
    `rsiprtp::srtp` per the existing feature bullet.

13. **Regenerate `public-api.txt`.** Per the comment in the file:
    `RUSTUP_TOOLCHAIN=nightly cargo public-api --manifest-path crates/rsiprtp/Cargo.toml --color never > crates/rsiprtp/public-api.txt`.
    Expect a large but mechanical diff.

14. **Run the validation matrix.** See below.

## Quality completion (in scope)

The collapse alone leaves the tree publishable but with five known quality
gaps, all of which must close before publish. Sequence them after the
mechanical collapse so each runs against a stable baseline:

15. **Fix the three `clippy::collapsible-match` errors in
    `rsiprtp/src/dialog/`** (formerly `rsiprtp-dialog`). Mechanical lint
    fix; bounded scope.

16. **Fix `registration::test_needs_refresh` on Windows.** The test panics
    in `std::time` arithmetic — almost certainly an `Instant - Duration`
    underflow when the registration's `last_refresh` is at-or-near the
    test's `Instant::now()`. Saturating subtraction or rebuilding the
    test around `Duration::checked_sub` resolves it. Bounded scope.

17. **Fix `test_generate_tag_unique` on macOS.** Generates N tags and
    asserts uniqueness; intermittently fails. Either the entropy source
    behaves differently on macOS under load, or the assertion's N is too
    large for a probabilistic uniqueness check. Read the test, raise the
    entropy, or shrink N. Bounded scope.

18. **MSRV `--locked` build under Rust 1.75.** `cargo build --workspace
    --exclude gabby --locked` fails on the MSRV job because `Cargo.lock`
    pins transitive deps that need newer rustc. Two paths:
    a) Identify the offending dep and pin a 1.75-compatible version in
       `[workspace.dependencies]`.
    b) Bump MSRV to whatever the lockfile actually requires and update the
       README and `rust-version` field.
    The collapse changes Cargo.lock substantially, so do this *after* the
    collapse, against the new lock. Bounded scope, but the answer (a vs b)
    depends on what the failing dep is — start with `cargo +1.75 build
    --locked 2>&1 | head` to identify it.

19. **Doc audit: re-tighten `#![warn(missing_docs)]` to `#![deny(...)]`.**
    Add doc comments to every undocumented `pub` item across the merged
    crate. **Open-ended scope** — could be 20 items or 200; we won't know
    until `cargo doc --no-deps 2>&1 | grep "missing documentation"` runs
    against the merged tree. Budget this as the largest piece of the
    quality work and don't be surprised if it's the longest commit.

20. **Visibility tightening.** Items that were `pub` solely to cross a
    former crate boundary, and are not part of the documented public
    surface (the prelude + items reachable as `rsiprtp::<module>::Item`
    that the README or examples actually use), can become `pub(crate)`.
    **Open-ended scope.** Use the regenerated `public-api.txt` as a
    starting point: anything in there that wasn't intentional public API
    is a candidate. Don't over-tighten — when in doubt, leave `pub`.

21. **Add `CHANGELOG.md`** at repo root. Use the [Keep a Changelog]
    (https://keepachangelog.com/) format. The first entry is `0.2.0 —
    2026-05-DD` covering this release: collapsed workspace into a single
    publishable crate, added SRTP/ICE to the public surface,
    `dtls` feature flag for opt-in DTLS-SRTP, dropped the `opus` feature
    (now always-on). Bounded scope. The README's earlier reference to a
    changelog (which we removed) can stay removed — readers will find
    `CHANGELOG.md` at the repo root the normal way.

22. **Confirm the version field.** `[workspace.package] version = "0.2.0"`
    matches the locked-in plan. The collapse is a structural change to an
    unpublished crate, so no semver concerns — 0.2.0 stays. Add a
    `[workspace.package] rust-version` bump here if item 18 forces an
    MSRV change.

Items 15–18, 21, 22 are bounded and short. Items 19–20 are the wildcards.
If 19 or 20 reveal substantial work, finish the collapse and the bounded
items first (commit), then tackle each open-ended item in its own commit
on the same branch. Atomicity at merge, reviewability per commit.

## Edge cases / risks

- **`crate::` references inside re-rooted modules.** Already covered by
  step 3 of mechanics. Mistakes produce compile errors, not silent
  breakage — the compiler catches them all.

- **`Cargo.lock` churn.** Substantial. Expected; commit it. The
  multi-version warnings in `cargo deny check` may shift as transitive
  resolution moves around — re-run and update the audit trail in
  `deny.toml` if anything changes.

- **Visibility tightening risk of over-correction.** Marking something
  `pub(crate)` that an external user (or `gabby`, or `fuzz`) actually
  imports causes a build break. Run all three (`cargo build -p rsiprtp`,
  `cargo build -p gabby`, `cargo build --manifest-path fuzz/Cargo.toml`)
  after each batch of tightening, not just at the end.

- **Doc audit may surface real API design questions** (an item with no
  obvious doc comment may be a sign it shouldn't be `pub` in the first
  place). When that happens, prefer `pub(crate)` over inventing a doc
  comment. The two passes (visibility, docs) interact; do visibility
  first, then docs against the smaller surface.

## Out of scope

- Splitting integration tests into smaller files.
- Adding a `CHANGELOG.md`.
- Bumping the `package.version` past `0.2.0` (already locked in).

## Validation

The change is done when, in this order:

1. `cargo build --workspace --exclude gabby` — succeeds.
2. `cargo build --workspace --exclude gabby --features dtls` — succeeds
   (gates the openssl-dependent SRTP/DTLS code).
3. `cargo test --workspace --exclude gabby -- --test-threads=1` passes
   on Linux, macOS, **and** Windows. (The two flakes from items 16–17
   must be fixed.)
4. `cargo clippy --workspace --exclude gabby --all-targets -- -D warnings`
   passes on stable rustc — no exceptions. (Item 15.)
5. `cargo +1.75 build --workspace --exclude gabby --locked` succeeds, or
   MSRV has been formally raised in the workspace and README. (Item 18.)
6. `cargo doc -p rsiprtp --no-deps --all-features` builds with no
   warnings, and `#![deny(missing_docs)]` is back on the crate. (Item 19.)
7. `cargo deny check` passes. (Expect to simplify the `wildcards = "warn"`
   rationale comment in `deny.toml` since the "single-facade with
   path-only deps" no longer applies.)
8. `cargo build -p gabby` (with `VOSK_LIB_DIR` set) still builds.
9. Fuzz harness still compiles:
   `cargo build --manifest-path fuzz/Cargo.toml`.
10. **`cargo publish --dry-run -p rsiprtp` succeeds** — the real exit
    gate. It's what crates.io will run.

No staged rollout, no migration, no compatibility shims. One branch,
several commits, one merge.
