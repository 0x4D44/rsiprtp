## Deviations from the plan

- **`fuzz/Cargo.toml` — dropped `features = ["dtls"]`.** The HLD says the
  `dtls` feature "is needed for `srtp_use_ext.rs`". Verified against the actual
  source: `parse_use_srtp_extension` in `srtp/dtls.rs` is not gated by any
  `#[cfg(feature = "dtls")]`. The whole `srtp::dtls` module is currently
  unconditional Rust code (no openssl call sites yet); the `dtls` feature only
  flips the optional `openssl` dep on. So the fuzz target compiles fine
  without the feature. Keeping it on would force an `openssl` system
  dependency on every fuzz build host with no benefit. The flag stays defined
  on the published crate (per HLD point 3) — the deviation is only on
  `fuzz/Cargo.toml`.
- **`crates/rsiprtp-transport/` empty dir not removed.** Windows held a
  filesystem lock on this empty directory after `git rm -r`; the index
  correctly drops the directory, and the directory is empty, but `rmdir`
  reports "Device or resource busy" (likely Windows Search indexer or AV).
  Functionally inert: Cargo doesn't reference it, git doesn't track it.
  Will retry from a clean shell post-commit; if persistent, schedule for the
  next reboot.
- **Hand-written SIP fuzz corpus discarded.** `crates/rsiprtp-sip/fuzz/corpus/parse_sip_message/`
  contained ten hand-crafted RFC-3261 messages. The HLD's "Delete the 11 empty
  crate dirs" doesn't address the corpus, but step 4 only mentioned rewiring
  the top-level `fuzz/` Cargo.toml. The same target (`sip_message`) exists at
  the workspace level without a seed corpus, so this is a real (if minor)
  loss. Out of scope to migrate it; flagged for follow-up.
