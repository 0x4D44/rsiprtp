- 2026-05-02: Apply-feedback agent surfaced a structural gap: the
  existing `IceAgent` is **send-only** — it has no recv loop on the
  bound sockets, so it cannot answer incoming STUN binding requests.
  Two `IceSession`s on loopback can never reach `PairState::Succeeded`
  against each other. The current loopback tests use a `#[cfg(test)]
  inject_validated_for_test` seam to wire (socket, peer) the way the
  success path would. This is a workaround, not a solution.
  Inserting Phase 2.5 — STUN responder on `IceAgent` — before Phase 3
  so the integration test in Phase 4 can be a real end-to-end oracle
  rather than relying on the test seam.
