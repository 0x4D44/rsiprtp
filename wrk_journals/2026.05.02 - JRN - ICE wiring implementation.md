  read only by `Default`, never wired into the gather path.
  Reviewer caught: missing `public-api.txt` refresh, silent no-op on
  bad media_index, session-level `c=` over-stepping, missing
  non-ICE-with-rtcp-mux test, multi-m-line rejection leaking
  offerer attrs.
- 2026-05-02: Phase 1 apply-feedback complete. All 7 required + 3
  suggested items applied. `gather_timeout_ms` now wraps
  `gather_candidates` in `tokio::time::timeout` (returns partial
  results on timeout). Test count: 2577 → 2582 lib tests passing.
  clippy clean, fmt clean, public-api.txt regenerated.
- 2026-05-02: Phase 2 IceSession implemented. 2582 → 2586 lib tests.
  Devils-advocate caught a correctness defect: `run_checks` was
  reporting `Ok` for pairs the agent picked via `fallback_selection`
  without any STUN check actually succeeding. Fixed by enforcing
  `pair.state == PairState::Succeeded`.
- 2026-05-02: Phase 2 apply-feedback complete. Validated-pair guard
  in. `default_candidate` now picks first non-loopback host
  deterministically. Failure test now exercises real STUN-timeout
  path. 2586 → 2588 lib tests.
- 2026-05-02: Apply-feedback agent surfaced a structural gap: the
  existing `IceAgent` is **send-only** — it has no recv loop on the
  bound sockets, so it cannot answer incoming STUN binding requests.
  Two `IceSession`s on loopback can never reach `PairState::Succeeded`
  against each other. The current loopback tests use a `#[cfg(test)]
  inject_validated_for_test` seam to wire (socket, peer) the way the
  success path would. This is a workaround, not a solution.
  Inserting Phase 2.5 — STUN responder on `IceAgent` — before Phase 3
  so the integration test in Phase 4 can be a real end-to-end oracle
  rather than relying on the test seam.
- 2026-05-02: Phase 2.5 implementation done. Per-socket STUN
  dispatcher: single recv loop demuxes by message type, validates
  USERNAME (after MI per RFC 5389 §10.1.2) and MESSAGE-INTEGRITY,
  responds with XOR-MAPPED-ADDRESS. Connectivity-check path
  refactored to register a oneshot in `pending_responses` keyed by
  transaction ID before sending. Test seam removed.
- 2026-05-02: Phase 2.5 review caught:
  - MI/USERNAME validation order wrong.
  - Attribute walk continued past MI.
  - Probe-stealing — the dispatcher reads continuously, so RTP
    sent on the same socket would be consumed by the dispatcher.
    The original loopback test had to `close()` before sending
    the probe — a tombstone for the design.
  - Behaviour regression: malformed STUN responses silently
    became `Timeout` instead of `InvalidResponse`.
  - Symmetric-NAT prflx unhandled (out of scope per HLD —
    documented).
- 2026-05-02: Phase 2.5 apply-feedback complete. Added
  `IceAgent::stop_responders()`; `IceSession::run_checks` calls it
  on success so the dispatcher stops reading from the bound sockets
  before the app takes over for RTP. Loopback probe test now sends
  through the live, post-success socket without any `close()` hack
  — this is the real oracle the HLD demanded. Dispatcher now
  forwards by txn-id (preserves `InvalidResponse` semantics).
  IPv6 XOR-MAPPED-ADDRESS implemented. 2588 → 2592 lib tests.
- Future-work notes (out of HLD scope): symmetric-NAT prflx
  discovery (RFC 8445 §7.3.1.4); parallel ICE checks (RFC 8445
  §6.1.4.2 — currently sequential which is pathological for many
  candidate pairs); RFC 7675 consent freshness keepalives during
  long calls.
- 2026-05-02: Phase 3 implementation done. `accept_inbound_invite`
  + `build_answer_for` (now takes `&IceAnswerInputs`) + factored
  shared helper. 2592 → 2597 lib tests.
- 2026-05-02: Phase 3 review caught: stuck-call (no reject path
  for Ringing), double-negotiation (two `create_answer` calls per
  inbound ICE call), dead empty-media check, ergonomic API
  (separate `default_candidate` and `ice_local` invite mismatch),
  pending-state fields could be hidden behind a constructor.
  Devils-advocate agent ran out of usage; proceeded with the
  regular review's findings plus a supervisor-added "full
  happy-path unit test" item.
- 2026-05-02: Phase 3 apply-feedback complete. Added
  `reject_inbound_invite`. `accept_inbound_invite` runs
  `create_answer` exactly once and caches `PendingAnswer { offer,
  negotiated }`. New `IceAnswerInputs<'a>` struct so callers
  can't pass mismatched candidate/local. Pending state hidden
  behind `Call::new_inbound_pending` ctor. 2597 → 2602 lib tests.
