## Fix-by-fix audit

### 1 + 2. `while !buf.len().is_multiple_of(4)` → `while buf.len() % 4 != 0`

`rtcp.rs` lines 559 and 668. Both loop bodies unconditionally call
`buf.put_u8(0)`, which grows `BytesMut` length by exactly 1 per iteration.
Therefore:

- **Termination:** at most 3 iterations before `len % 4 == 0`. No infinite-loop
  risk regardless of starting length.
- **`usize::MAX` corner:** `buf` is `BytesMut`, bounded by `usize::MAX` and
  available memory. For any valid `usize`, `n % 4` and `n.is_multiple_of(4)`
  agree byte-for-byte (`usize::MAX % 4 == 3` on 64-bit, so the loop would
  iterate, but you'd be OOM long before reaching that state).
- **Codegen:** `is_multiple_of(4)` for power-of-two divisors lowers to the same
  bitwise-AND test as `% 4`. LLVM optimizes both identically; no inner-loop
  perf regression. (Even if it didn't, this is RTCP packet build — not a hot
  path.)
- **Comments / asserts dropped:** none. Diff is line-for-line.
- **`#[cfg(test)]` invariants:** the SDES roundtrip and Goodbye build paths in
  `mod tests` exercise these loops; they don't depend on the precise predicate
  spelling.

Verdict: **safe**.

### 3. `Duration::abs_diff(rtt)` → manual `if` branch

`session.rs:223–227` in `CongestionController::on_rtt`.

```rust
let diff = if self.smoothed_rtt > rtt {
    self.smoothed_rtt - rtt
} else {
    rtt - self.smoothed_rtt
};
```

- **Equality case:** original `abs_diff` returns `Duration::ZERO`.
  Manual: `>` is strict, equal values fall to else, computing `rtt - rtt = 0`.
  Equivalent — and importantly safe (subtracting equals never panics).
- **Sub-microsecond / nanosecond boundary precision:** `Duration` stores
  `(secs: u64, nanos: u32)`. Both `abs_diff` and manual subtraction operate on
  this representation; there is no float arithmetic. Either branch produces
  the exact non-negative difference. No precision loss vs. `abs_diff`.
- **Overflow / panic:** the manual rewrite picks the larger operand for the
  minuend in both branches, so subtraction never underflows. Same guarantee
  as `abs_diff`. No new panic surface.
- **Surrounding logic unchanged:** subsequent `self.rtt_var = self.rtt_var * 3 / 4 + diff / 4`
  uses `Duration` arithmetic that is itself stable since 1.0/1.3.

Verdict: **safe**. Note the manual rewrite is slightly noisier; if MSRV is
ever bumped past 1.81, reverting to `abs_diff` would be a clarity win, but
that's a future tidy-up, not a blocker.

### 4. `Box::new(G722Codec::new())` → `Box::default()`

`session_codec.rs:51`. Read `media/codec/g722.rs:97–101`:

```rust
impl Default for G722Codec {
    fn default() -> Self { Self::new() }
}
```

- `Box::default()` desugars to `Box::new(<T as Default>::default())`, which is
  `Box::new(G722Codec::new())`. Allocation count, ordering, and side effects
  are identical (one heap allocation, after the inner `new()` builds the
  encoder/decoder pair).
- `G722Codec::new()` initialises real codec state (encoder + decoder at
  `Bitrate::Mode1_64000`). That state is built before the `Box::new` allocation
  in both forms — no observable difference.
- Type inference: the surrounding pattern `SessionCodec::G722(Box::default())`
  drives the boxed type from the `G722` variant's payload, so this compiles
  cleanly without turbofish.

Verdict: **safe**. (Stylistically: clippy actually *prefers* this form — the
`box_default` lint flags `Box::new(T::default())`. So this fix not only doesn't
introduce concerns, it eliminates a future lint trip.)

### 6. New MSRV regressions introduced by the fixes?

Checked replacements:

- `n % 4 != 0` — core `Rem` for `usize`, stable since 1.0.
- `if a > b { a - b } else { b - a }` — `Duration: Sub<Duration>` and `PartialOrd`,
  stable since 1.3.
- `Box::default()` — stable since 1.0.

None of the replacements introduces a newer-than-1.75 API. Codebase scan for
`is_multiple_of` / `Duration::abs_diff` shows no remaining occurrences (only
journal mentions and the unrelated `u32::abs_diff` in
`bitrate_bridge.rs:97`, which is fine on 1.60+).
