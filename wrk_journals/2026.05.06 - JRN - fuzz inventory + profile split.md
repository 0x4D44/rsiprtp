## Goal

Two work items, in order:

1. **Inventory test** — a Rust test that walks `fuzz/fuzz_targets/*.rs` +
   `crates/rsiprtp/fuzz/fuzz_targets/*.rs` and the `*.ps1` wrapper files,
   asserts every target file is referenced by at least one wrapper. Prevents
   "added a fuzz target, forgot to schedule it" drift permanently. Runs as
   part of `cargo test --workspace`, so it's enforced by `full_test`.
2. **Wrapper profile-split** — refactor the rotation so we have multiple
   profiles (sip, rtp, media, ice). Each profile gets its own night.
   Drives the previously-dormant RTP/RTCP/SRTP/codec/ICE targets into
   actual fuzz time. The inventory test is the spec for "did we cover
   everything."

zero-data-loss default. Retirement of predecessor targets (`sip_message`,
`sip_via`, `sdp_session` — superseded by their `_roundtrip`/`_typed`
variants) can happen as a separate change once the new profiles have
proven out.

### Stage 3 — Wrapper profile-split (complete)

`fuzz_overnight.ps1` refactored:
- `-Profile <name>` parameter (default `sip`).
- `$Profiles` hashtable defines six profiles (sip, sip-legacy, sip-diff,
  rtp, media, ice).
- `$TargetCwd` auto-discovered from filesystem: targets in
  `fuzz/fuzz_targets/` → cwd = repo root; targets in
  `crates/rsiprtp/fuzz/fuzz_targets/` → cwd = `crates/rsiprtp`. Adding
  a target file no longer requires a wrapper edit for cwd lookup.
- `-Targets` override still works for ad-hoc runs.
- `START` event grew a `profile=<name>` field.
- Unknown profile name throws with a sorted list of known profiles.
