## Plan

I'm orchestrating agents through these stages, committing after each:

1. **Stage 1 — Implementation.** One agent writes the scaffold + all
   passes + report generation + main(), with internal unit tests
   where they pull weight (parsers, path-collision, categorizer).
   Verifies `cargo build -p full_test` and `cargo run -p full_test
   --skip-coverage` runs end-to-end on this tree.
2. **Stage 2 — Code review.** code-reviewer agent reviews the diff
   against rust-best-practices and the V2 HLD. Then a fixer agent
   applies the actionable feedback (code-review-apply-feedback).
3. **Stage 3 — Devil's advocate.** Independent agent reads the V2
   HLD and the diff cold and challenges scope/correctness/edge
   cases. Fixer agent addresses anything material.
4. **Stage 4 — Ancillary deliverables.** README, CLAUDE.md,
   `.gitignore`, `.gitkeep` (anything Stage 1 didn't cover).
5. **Stage 5 — Completeness pass.** Compare implementation against
   V2 HLD line by line. Iterate fixes until 100% match.

## Ground rules I'm holding the agents to

- The runner is a dev tool — keep it small. No premature abstractions.
- Match the project's CI command idioms (`--workspace --exclude gabby
  --exclude full_test`, `RUSTDOCFLAGS=-D warnings`, `--test-threads=1`).
- `chrono` only as a third-party dep. HTML escape and JSON scrape
  inline (~30 LoC each, copy from mddosem).
- Every cargo invocation excludes `gabby` and `full_test` to dodge
  the Vosk dep and the Windows self-rebuild lock.
- Optional tools (`cargo-deny`, `cargo-audit`, `cargo-llvm-cov`)
  probed with `--version`; missing → warn + skip, do not fail.
- One HTML report per run, dated path with `(N)` collision suffix.
- Only `chrono`, no `tempfile`, no `colored`, no `base64` (no
  screenshots).

### 2026.05.02 — Stage 2 complete

Code review found nothing blocking. Six suggestions, all worth doing —
applied them in one pass:

1. **Coverage exit-code gap.** `CoverageResult` became an enum
   (`Skipped` / `Failed { stderr_tail }` / `Ok(metrics)`). A genuine
   `cargo llvm-cov` crash now flips overall exit code and renders a
   Coverage FAIL section. Previously a coverage crash was silently
   indistinguishable from "tool not installed".
2. **JSON scrape replaced with serde_json.** `extract_json_number`
   deleted; `parse_coverage_json` now uses `serde_json::Value`
   lookups against `data[0].totals.<section>.<key>`. `serde_json`
   added as a dep (workspace inheritance). Tests adapted, plus three
   new ones including the nested-subobject case that was the
   original concern.
3. **Quality-check duplication.** ~120 LoC of scaffolding collapsed
   into a single `run_quality_check(name, args, is_issue)` helper
   with a `skipped_quality` constructor. The four `run_*` quality
   functions are now 3–7 line wrappers.
4. **Dead `process_failure` field** removed.
5. **Redundant `tool_available("llvm-cov")`** call in `main` removed
   (kept the canonical site inside `run_coverage`).
6. **`run_doc` stderr retention** gated on `!passed`.

Net: +330 / −249, 13 unit tests (was 8), all clean under fmt + clippy
+ build. End-to-end smoke produced a report at the documented path
with the new Coverage FAIL section visible.
