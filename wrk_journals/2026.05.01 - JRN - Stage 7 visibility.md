# Stage 7 — Visibility tightening

**Date:** 2026-05-01
**Author:** Claude (Stage 7 implementer)
**HLD:** `wrk_docs/2026.05.01 - HLD - Collapse workspace into single rsiprtp crate.md` (item 20)

## Goal

Tighten visibility on items that became reachable as `rsiprtp::<module>::Item`
purely as a side effect of the workspace collapse. The pre-collapse pattern
of `pub` everywhere-that-crossed-a-crate-boundary became leaky once the
boundaries dissolved.

## Approach

Worked at two granularities:

1. **Submodule level (high leverage).** Many former-crate `mod.rs` files
   declared every internal source file as `pub mod foo;`. After grepping
   `rsiprtp::<module>::<sub>::` across `crates/`, `examples/`, `fuzz/`,
   and `gabby/`, found which submodules genuinely needed external
   reachability. Tightened the rest to `pub(crate) mod`.

2. **Item level (targeted).** A handful of free helper functions inside
   former-crates (`sdp_direction_attribute`, `parse_sdp_direction`,
   three `build_refer_*` helpers, G.722 `silence_frame`, Opus
   `OPUS_FRAME_MS`/`OPUS_MAX_PACKET_SIZE`, `core::random_u{16,32,64}`)
   were `pub` only for cross-crate access — exercised purely by their
   own tests. Moved to `pub(crate)` (gated behind `#[cfg(test)]`
   where they had no non-test caller).

|---|---|---|

| `sip` | `auth`, `headers`, `message`, `uri` | — |
| `transport` | `resolver`, `tcp`, `tls`, `traits`, `udp` | — |
| `dialog` | `invite`, `manager`, `state` | — |
| `transaction` | `manager` | — |
| `rtp` | `dtmf`, `packet` | — |
| `ice` | `agent`, `candidate`, `stun`, `turn` | — |
| `srtp` | (none — `dtls` reached by fuzz) | — |
| `sdp` | (none — `builder`, `negotiation`, `parser` all reached externally) | — |
| `media` | `codec` (cascades to `g711`/`g722`/`opus`) | G.722 `silence_frame`, Opus `OPUS_MAX_PACKET_SIZE`; deleted unused `OPUS_FRAME_MS`; removed redundant re-exports in `codec/mod.rs` |
| `session` | `bitrate_bridge`, `hold`, `manager`, `registration`, `session_codec`, `transfer` (kept `call` because tests reach `session::call::MediaSession`) | 5 free helper functions (`sdp_direction_attribute`, `parse_sdp_direction`, `build_refer_to_blind`, `build_refer_to_attended`, `build_referred_by`) |

**Rough count:** 27 submodules tightened + ~10 item-level changes. Each
submodule cascades to all its `pub` items, so the actual API-surface
reduction is much larger than 37: the regenerated `public-api.txt` shrank
from ~25,237 lines to **18,925 lines** — roughly **25% of the published
API surface eliminated**.

## What stayed `pub`

Everything reached from outside the crate by:

- `crates/rsiprtp/examples/basic_call.rs` — uses `prelude::*` plus
  `rsiprtp::sdp::builder::MediaBuilder`.
- `crates/gabby/src/server.rs` — uses `rsiprtp::sip::{generate_tag, Method,
  SipMessage, SipRequest, SipResponse, SipResponseBuilder}`.
- `crates/gabby/tests/media_branch_coverage.rs` — uses
  `rsiprtp::media::{generate_dtmf_tone, is_silence, WavReader}`.
- `fuzz/fuzz_targets/*.rs` — reach `rsiprtp::rtp::rtcp::*`,
  `rsiprtp::srtp::dtls::parse_use_srtp_extension`, top-level types in
  `rtp`, `sip`, `sdp`, `ice`, `srtp`.
- Internal integration tests under `crates/rsiprtp/tests/` — reach
  `rsiprtp::rtp::rtcp::*`, `rsiprtp::rtp::session::CongestionController`,
  `rsiprtp::sdp::negotiation::Codec`, `rsiprtp::sdp::parser::*`,
  `rsiprtp::session::call::MediaSession`,
  `rsiprtp::transaction::{client, server, timer}::*`,
  `rsiprtp::sip::DigestCredentials`/`DigestResponse`/`SipResponse`.

These dictated the pub/pub(crate) split. (Internal tests count as
consumers because they're inside the crate's own tarball; making the
items they reach `pub(crate)` is fine for that purpose, but several of
them are tests in the `tests/` directory, which compile as separate
crates and thus need top-level reachability.)

## Skipped / deferred

- **`media::codec::AdaptiveBitrate` trait.** Currently re-exported as
  `media::AdaptiveBitrate` and trafficked via `SessionCodec`'s sum-type
  dispatch. Left `pub` — it's part of the documented adaptive-codec API.
- **`session::call`** kept `pub` because `tests/rtcp_drives_bridge.rs`
  reaches `session::call::MediaSession` directly. Could in principle be
  collapsed via a top-level re-export in `session/mod.rs` and then
  `pub(crate) mod call`, but the prelude already exports `MediaSession`
  via `session::*`, so this is bikeshedding.
- **Per-field visibility.** `rtp::ReceiverState` has all-`pub` fields and
  is returned by `RtpSession::receiver_stats`. Tightening the fields
  to add accessor methods is a real API change with semver implications;
  out of scope for visibility-only sweep.
- **`rtp::session::CongestionState`/`CongestionController`** kept `pub`.
  Only used in tests externally, but they're documented in the bitrate-
  bridge HLDs as the AIMD primitive — a reasonable consumer might want
  to drive congestion control directly.

## Reversions during the sweep

None. Three or four builds caught dead-code warnings (G.722
`silence_frame`, Opus `OPUS_FRAME_MS`/`OPUS_MAX_PACKET_SIZE`, the four
session free helpers, and the codec re-exports inside
`media/codec/mod.rs`). All were addressed inline either by gating
behind `#[cfg(test)]` (test-only callers), promoting to `pub(crate)`
(internal callers), or deleting outright (`OPUS_FRAME_MS` had no callers
at all).

The fuzz target `srtp_use_ext` reaches
`rsiprtp::srtp::dtls::parse_use_srtp_extension` — that single item kept
`srtp::dtls` from being demoted. The other two `pub fn` items in
`srtp/dtls.rs` (`build_use_srtp_extension`, `extract_srtp_keys`) appear
in module docs as part of the documented DTLS-SRTP API; left `pub`.

## Surprises

- The `media::codec::g711` re-exports of `silence_byte`, `silence_frame`,
  `G711Codec`, `G711Variant` happened in TWO places: the `codec/mod.rs`
  shim AND `media/mod.rs`. After demoting `codec` to `pub(crate)`, the
  shim re-exports in `codec/mod.rs` were dead — `media/mod.rs` reaches
  through to `codec::g711::*` directly. Removed the redundant re-exports.
- Looking at the regenerated `public-api.txt` shows the visibility shrink
  almost entirely came from collapsing `media::codec::{g711,g722,opus}`
  out of the public API — those three sub-submodules dump enormous trait
  impl listings for every codec type. Single biggest line-count win.
- `transport::resolver` had a doc comment with
  `use rsiprtp::transport::resolver::{...};` — after the demotion the
  doc would have been broken if anyone exercised it. Updated to
  `use rsiprtp::transport::{...};` in the same pass.
