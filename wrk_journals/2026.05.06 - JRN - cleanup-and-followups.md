- **Never delegate the long wait.** If any agent kicks off a
  long-running build/test, supervisor drives the wait via Bash
  `run_in_background` + the agent reports the command, not the
  outcome. (Per `feedback_supervised_subagent_waits.md`.)
- **HLDs first, code second.** Implementation agents do not start
  until the design is signed off. Reduces wasted compute and keeps
  the human-in-the-loop at the cheap stage.
- **One HLD per task.** No unified design doc; each follow-up
  travels on its own merits.
- **Journal cadence:** entry per phase boundary, plus any
  notable mid-phase event (review-blocking finding, agent stalled,
  etc.). Saved to disk via `Write`/`Edit`, not buffered.
