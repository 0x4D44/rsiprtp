  Artifacts dir is empty — no known-bad inputs at start.
- Single fuzz target today: `sip_message_parse` (only one in tree).

## Run config

- `cargo +nightly fuzz run sip_message_parse --fuzz-dir crates/rsiprtp/fuzz`
- libFuzzer args:
  - `-max_total_time=28800` (8 h per worker)
  - `-workers=4 -jobs=4` (4 parallel)
  - `-rss_limit_mb=2048`
  - `-print_final_stats=1`
- stdout+stderr → `crates/rsiprtp/fuzz/run.log`
- Detached via `Start-Process` so it survives parent shell churn.

## Supervision plan

- Wake every ~25-30 min to:
  1. Tail `run.log` for crash/leak/oom signatures
  2. `ls artifacts/sip_message_parse/` for new findings
  3. Update journal with exec count, exec/s, corpus growth
  4. If artifact found: dispatch triage agent, then a fix/review agent
- All triage/fix work happens in subagents to keep this context lean.
